# Продуктовое программирование 2026 — Геймификация туризма в регионе

Проект для **Чемпионата по продуктовому программированию Рязанской области 2026**.  
Команда: **Код и Кофе**.

## Кейc
**Геймификация туризма в регионе** — разработка инновационного сервиса для вовлечения туристов через игровые механики и QR‑квесты.

### Проблема
Почему традиционный туризм теряет вовлеченность?
- Отсутствие игровых механик, стимулирующих повторные посещения.
- Скучные форматы классических экскурсий без эмоционального погружения.
- Низкая мотивация молодежи исследовать локальные достопримечательности без интерактивного слоя.

### Пользовательский путь
1. Выбор и покупка маршрута.
2. Поиск QR‑кодов на маршруте.
3. Сканирование и подтверждение посещений.
4. Получение баллов.
5. Обмен баллов на награду.

## Функциональные требования (Must Have)
- Интерактивная карта (POI, маршруты, текущее местоположение).
- Сканер QR и профиль.
- Баланс бонусных баллов.
- История пройденных квестов.

## Nice to Have
- Лидерборды, командные забеги и обмен достижениями.
- Интеграция с партнерами (кофейни, музеи, транспорт).
- ИИ‑гид для ответов на вопросы о достопримечательностях.

# Архитектура (MVC)
**Backend (Django):**
- `quests/models.py` — модели (данные).
- `quests/views/*` — контроллеры по доменам (`auth`, `poi`, `progress`).
- `quests/serializers.py` — транспортные DTO.

**Frontend (React):**
- `frontend/src/data/*` — модели/константы.
- `frontend/src/components/*` — представления.
- `frontend/src/hooks/*` — контроллерная логика (например, прогресс).
- `frontend/src/services/*` — API‑доступ.

# Запуск проекта (Windows / Linux)

## 1. Backend (Django)
### Установка зависимостей
```bash
python -m venv venv
source venv/bin/activate            # Linux/Mac
venv\\Scripts\\activate             # Windows
pip install -r requirements.txt
```

### База данных
По умолчанию используется PostgreSQL (см. `core/settings.py`).
Проверь `.env`:
```
DB_NAME=tourism
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
```

### Миграции и запуск
```bash
python manage.py migrate
python manage.py runserver
```

## 2. Frontend (Vite + React)
```bash
cd frontend
npm install
npm run dev
```

Опционально в `frontend/.env`:
```
VITE_API_BASE=http://localhost:8000
```

# Развёртывание (хостинг)
**Вариант 1: фронт и бэк отдельно**
1. Сборка фронта:
   ```bash
   cd frontend
   npm install
   npm run build
   ```
2. Залить `frontend/dist` на статический хостинг (Netlify/Vercel/Cloudflare Pages).
3. На бэке:
   - `DEBUG=False`
   - `ALLOWED_HOSTS=ваш-домен`
   - `FRONTEND_URL=https://ваш-фронт`
4. Запуск Django через `gunicorn` + `nginx`.

**Вариант 2: всё на одном сервере**
- `nginx` раздает `frontend/dist` и проксирует `/api` на Django.
